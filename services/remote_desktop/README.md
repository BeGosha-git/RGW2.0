# Удалённый стол (Remote Desktop)

Схема: **ПК (хост) ↔ сервер ↔ ПК (клиент)**. Весь трафик идёт через сервер. Хосты сами регистрируются на сервере; клиент подключается к серверу, получает список ПК и выбирает, к какому подключаться.

## Компоненты

| Компонент | Запуск | Описание |
|-----------|--------|----------|
| **Сервер** | Только вручную | Реле на порту 9009. Не запускается из RGW. |
| **Хост** | Сервис RGW или вручную | Регистрируется на сервере, отдаёт экран(ы), принимает мышь/клавиатуру. |
| **Клиент** | Только вручную | Подключается к серверу, показывает список ПК, после выбора — экран и управление. |

## Настройка

1. **Сервер**  
   Запуск вручную (на машине с известным IP):
   ```bash
   python -m services.remote_desktop.remote_desktop_server
   # или с портом: python -m services.remote_desktop.remote_desktop_server 9009
   ```
   Порт по умолчанию: **9009**.

2. **Параметры сервиса «remote_desktop» (хост)**  
   В веб-интерфейсе RGW → Сервисы → remote_desktop:
   - **server_host** — IP сервера (обязательно).
   - **server_port** — порт (по умолчанию 9009).
   - **pc_name** — имя ПК в списке (по умолчанию hostname).
   - **wake_password** — пароль для пробуждения (по умолчанию 1055, Ubuntu 24).

3. **Клиент**  
   На управляющем ПК:
   ```bash
   python -m services.remote_desktop.remote_desktop_client <IP_сервера> [порт]
   # пример: python -m services.remote_desktop.remote_desktop_client 192.168.1.100 9009
   ```
   Если IP не указан, запросится диалог.

## Функционал

- **Экран**: демонстрация всех мониторов хоста (один общий скриншот).
- **Мышь**: движение, клики, прокрутка.
- **Клавиатура**: нажатия и отпускания клавиш.
- **Сон (Ubuntu 24)**: пока хост подключён к серверу, переход в сон блокируется (`systemd-inhibit`). Пароль **1055** используется в параметре **wake_password** и при необходимости для WoL.
- **Пробуждение (WoL)**: если хост при регистрации передал MAC, сервер сохраняет его. После отключения хоста клиент может отправить запрос на пробуждение — сервер отправит Wake-on-LAN magic packet (порт 9).

## Зависимости

Установка для хоста и клиента:
```bash
pip install mss Pillow pynput
```
Или из корня проекта: `pip install -r requirements.txt`

## Важно

- Сервер **никогда** не запускается автоматически RGW — только вручную.
- Клиент тоже запускается только вручную на той машине, с которой вы управляете.
- Для пробуждения по WoL в BIOS/системе должен быть включён Wake-on-LAN; пароль 1055 при необходимости настраивается в BIOS (SecureOn).
